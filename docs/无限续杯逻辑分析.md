# 原始 Bash 脚本无限续杯（定时任务）逻辑分析

## 概述

无限续杯功能通过 cron 定时任务实现自动化的账号续期管理，核心文件：
- `unix/无限续杯.sh` - 入口脚本（18 行）
- `common/infinite_refill_loop_impl.sh` - 主实现（617 行）
- `unix/_定时任务_入口.sh` - Cron 任务入口（31 行）
- `unix/_内部_自动清理.sh` - 自动清理入口（19 行）

## 架构设计

### 整体流程

```
用户交互菜单
    ↓
配置管理（无限续杯配置.env）
    ↓
Cron 任务管理
    ↓
定时任务入口（_定时任务_入口.sh）
    ↓
    ├─→ 自动清理（_内部_自动清理.sh）
    │       ↓
    │   清理失效/过期账号
    │
    └─→ 单次续杯（单次续杯.sh --from-task）
            ↓
        增量闭环续杯
            ↓
        失败处理（退出码 4/5 自动停用）
```

## 核心模块分析

### 1. 定时任务入口（_定时任务_入口.sh）

#### 文件锁机制（第 8-18 行）

```bash
LOCK_FILE="$SCRIPT_DIR/_task.lock"
MAX_LOCK_AGE=600  # 10 分钟超时

# 检查锁文件
if [[ -f "$LOCK_FILE" ]]; then
  now="$(date +%s)"
  old="$(stat -c %Y "$LOCK_FILE" 2>/dev/null || stat -f %m "$LOCK_FILE")"

  # 如果锁文件未超时，直接退出（防止并发）
  if (( now - old < MAX_LOCK_AGE )); then
    exit 0
  fi

  # 超时则删除旧锁
  rm -f "$LOCK_FILE"
fi

# 创建新锁，退出时自动清理
trap 'rm -f "$LOCK_FILE" >/dev/null 2>&1 || true' EXIT
printf '%s\n' "$(date -u +%Y-%m-%dT%H:%M:%SZ)" > "$LOCK_FILE"
```

**关键点：**
- 防止并发执行（同一时间只能运行一个任务）
- 10 分钟超时保护（防止死锁）
- 使用 trap 确保锁文件清理

#### 串行执行流程（第 20-31 行）

```bash
# 1. 先执行自动清理（apply 模式）
bash "$SCRIPT_DIR/_内部_自动清理.sh" apply >/tmp/自动清理.log 2>&1 || true

# 2. 再执行单次续杯
set +e
bash "$SCRIPT_DIR/单次续杯.sh" --from-task >/tmp/无限续杯.log 2>&1
EC=$?
set -e

# 3. 失败自动停用
if [[ "$EC" == "4" || "$EC" == "5" ]]; then
  bash "$SCRIPT_DIR/无限续杯.sh" --disable-task-silent >/dev/null 2>&1 || true
fi

exit "$EC"
```

**执行顺序：**
1. **自动清理**：删除失效/过期账号，释放空间
2. **单次续杯**：增量闭环续杯，补充新账号
3. **失败处理**：退出码 4/5 时自动停用定时任务

**日志输出：**
- 自动清理日志：`/tmp/自动清理.log`
- 续杯日志：`/tmp/无限续杯.log`

### 2. Cron 任务管理（infinite_refill_loop_impl.sh）

#### 任务标识（第 191-212 行）

```bash
task_hash() {
  local uk="${1:-}"  # USER_KEY

  # 使用 USER_KEY 的 SHA256 前 6 位作为任务标识
  if command -v sha256sum >/dev/null 2>&1; then
    printf '%s' "$uk" | sha256sum | awk '{print substr($1,1,6)}'
  elif command -v shasum >/dev/null 2>&1; then
    printf '%s' "$uk" | shasum -a 256 | awk '{print substr($1,1,6)}'
  else
    printf '%s' "$uk" | openssl dgst -sha256 | awk '{print substr($NF,1,6)}'
  fi
}

# 生成任务标记
hash="$(task_hash "${USER_KEY:-}")"
marker="INFINITE_REFILL_${hash}"
```

**设计目的：**
- 支持多用户（不同 USER_KEY 生成不同标识）
- 防止任务冲突（每个用户独立的 cron 任务）
- 便于管理（通过标记识别和清理任务）

#### 启用定时任务（enable_task 函数，第 339-387 行）

```bash
enable_task() {
  # 1. 检查依赖
  ensure_runtime_base || return 1
  ensure_runtime_cron || return 1

  # 2. 加载配置
  load_cfg
  INTERVAL_MINUTES="$(sanitize_interval "${INTERVAL_MINUTES:-30}")"

  # 3. 生成任务标识
  hash="$(task_hash "${USER_KEY:-}")"
  marker="INFINITE_REFILL_${hash}"
  begin="# BEGIN_${marker}"
  end="# END_${marker}"

  # 4. 构建任务入口脚本
  build_task_entry  # 生成 _定时任务_入口.sh

  # 5. 读取当前 crontab
  cur="$(crontab -l 2>/dev/null || true)"

  # 6. 移除旧的任务块（如果存在）
  filtered="$(printf '%s\n' "$cur" | awk -v b="$begin" -v e="$end" '
    BEGIN { skip=0 }
    $0 == b { skip=1; next }
    $0 == e { skip=0; next }
    { if(skip==0) print }
  ')"

  # 7. 添加新的任务块
  cron_line="*/${INTERVAL_MINUTES} * * * * bash \"${TASK_ENTRY}\""
  {
    printf '%s\n' "$filtered"
    printf '%s\n' "$begin"
    printf '%s\n' "$cron_line"
    printf '%s\n' "$end"
  } | crontab -

  # 8. 清理其他用户的旧任务
  cleanup_old_tasks "$marker"

  echo "[OK] 已创建/更新定时任务：${REFILL_TASK_PREFIX}_${hash}（每 ${INTERVAL_MINUTES} 分钟）"
}
```

**Crontab 格式：**
```bash
# BEGIN_INFINITE_REFILL_a1b2c3
*/30 * * * * bash "/path/to/_定时任务_入口.sh"
# END_INFINITE_REFILL_a1b2c3
```

**关键特性：**
- 自动更新（重复调用会更新间隔时间）
- 多用户隔离（不同 USER_KEY 独立任务）
- 旧任务清理（防止重复任务）

#### 禁用定时任务（disable_task 函数，第 389-418 行）

```bash
disable_task() {
  local silent="${1:-0}"

  # 1. 生成任务标识
  hash="$(task_hash "${USER_KEY:-}")"
  marker="INFINITE_REFILL_${hash}"
  begin="# BEGIN_${marker}"
  end="# END_${marker}"

  # 2. 读取当前 crontab
  cur="$(crontab -l 2>/dev/null || true)"

  # 3. 移除任务块
  next="$(printf '%s\n' "$cur" | awk -v b="$begin" -v e="$end" '
    BEGIN { skip=0 }
    $0 == b { skip=1; next }
    $0 == e { skip=0; next }
    { if(skip==0) print }
  ')"

  # 4. 更新 crontab
  printf '%s\n' "$next" | crontab -

  [[ "$silent" != "1" ]] && echo "[OK] 已关闭定时任务：${REFILL_TASK_PREFIX}_${hash}"
}
```

#### 自动停用机制（第 527-551 行）

```bash
run_once() {
  set +e
  bash "$SCRIPT_DIR/单次续杯.sh"
  local ec=$?
  set -e

  if [[ "$ec" == "0" ]]; then
    # 成功：重置定时任务时间
    reset_task_after_manual
  elif [[ "$ec" == "4" ]]; then
    # 服务端禁用：自动关闭定时任务
    echo "[WARN] 服务端已自动禁用当前用户（超日限）。已自动关闭本机定时续杯任务。"
    disable_task 1
  elif [[ "$ec" == "5" ]]; then
    # 服务端封禁：自动关闭定时任务
    echo "[WARN] 服务端判定滥用并自动封禁。已自动关闭本机定时续杯任务。"
    disable_task 1
  fi

  return "$ec"
}
```

**退出码处理：**
- `0`：成功，重置定时任务时间
- `4`：服务端禁用（auto_disabled=true），自动停用
- `5`：服务端封禁（abuse_auto_banned=true），自动停用
- 其他：记录错误但继续运行

### 4. 同步目录功能（第 214-266 行）

```bash
ensure_sync_links() {
  local target="${SYNC_TARGET_DIR:-}"
  local accounts="${ACCOUNTS_DIR:-$ROOT_DIR/accounts}"

  # 如果未配置同步目录，跳过
  [[ -z "$target" ]] && return 0

  mkdir -p "$target" "$accounts"

  # 1. 清理已删除账号的软链接
  if [[ -f "$manifest" ]]; then
    while IFS= read -r base; do
      if [[ ! -e "$accounts/$base" && -L "$target/$base" ]]; then
        rm -f "$target/$base"
        removed=$((removed + 1))
      fi
    done < "$manifest"
  fi

  # 2. 创建新账号的软链接
  for f in "$accounts"/*.json; do
    base="$(basename "$f")"
    tp="$target/$base"

    # 如果是软链接，先删除
    if [[ -L "$tp" ]]; then
      rm -f "$tp"
    elif [[ -e "$tp" ]]; then
      # 如果是普通文件，跳过
      continue
    fi

    # 创建软链接
    ln -s "$f" "$tp"
    linked=$((linked + 1))
  done

  # 3. 更新清单文件
  printf "%s\n" "${names[@]}" > "$manifest"
}
```

**用途：**
- 与其他工具共享账号（如 cli-proxy-api）
- 使用软链接避免文件复制
- 自动同步账号变化

### 6. 依赖检查（第 79-131 行）

```bash
ensure_runtime_base() {
  # 1. 检查 curl
  if ! check_cmd curl; then
    echo "[ERROR] 缺少依赖：curl"
    return 1
  fi

  # 2. 检查 JSON 运行时（三选一）
  if ! has_json_runtime; then
    echo "[ERROR] 缺少 JSON 运行时：需要 jq 或 osascript 或 python3"
    return 1
  fi

  return 0
}

has_json_runtime() {
  local os="$(detect_os)"

  # macOS 优先使用 osascript
  if [[ "$os" == "macos" ]] && check_cmd osascript; then
    return 0
  fi

  # Linux 优先使用 jq
  if check_cmd jq; then
    return 0
  fi

  # 兜底使用 python3
  if check_cmd python3; then
    return 0
  fi

  return 1
}

ensure_runtime_cron() {
  if ! check_cmd crontab; then
    echo "[ERROR] 缺少依赖：crontab（定时任务功能不可用）"
    return 1
  fi
  return 0
}
```

**依赖层级：**
1. **必需**：curl + JSON 运行时（jq/osascript/python3）
2. **定时任务**：crontab
3. **可选**：其他工具

### 差异点

#### 1. 定时任务实现

**Bash：**
```bash

*/30 * * * * bash "/path/to/_定时任务_入口.sh"
```

**Go：**
```go

    runCleanAndRefill()

```

**优劣对比：**
- Bash：依赖系统 cron，更稳定，重启后自动恢复
- Go：进程内调度，更灵活，但需要进程保活

#### 2. 文件锁实现

**Bash：**
```bash
# 简单的文件时间戳检查
if [[ -f "$LOCK_FILE" ]]; then
  now="$(date +%s)"
  old="$(stat -f %m "$LOCK_FILE")"
  if (( now - old < MAX_LOCK_AGE )); then

  fi

```

**Go：**
```go
// 使用 flock 系统调用
lockFile, err := os.OpenFile(lockPath, os.O_CREATE|os.O_RDWR, 0644)
syscall.Flock(int(lockFile.Fd()), syscall.LOCK_EX|syscall.LOCK_NB)
```

**优劣对比：**
- Bash：简单但不够健壮（进程崩溃可能留下锁）
- Go：使用系统级锁，更可靠

#### 3. 配置管理

**Bash：**

**优劣对比：**
- Bash：交互式更友好（适合首次配置）

## 总结

### 核心设计原则

1. **串行执行**：先清理后续杯，避免资源竞争
2. **文件锁保护**：防止并发执行，10 分钟超时
3. **失败自动停用**：退出码 4/5 时自动关闭定时任务
4. **多用户隔离**：使用 USER_KEY hash 区分不同用户
5. **日志分离**：清理和续杯日志独立输出
