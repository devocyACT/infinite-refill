#### 并发控制（第 1472-1491 行）

```bash
# 启动 worker 进程
for f in "${PROBE_FILES[@]}"; do
  bash "$SCRIPT_DIR/单次续杯.sh" --probe-one-worker "$f" ... &

  # 控制并发数
  while true; do
    running="$(jobs -pr | wc -l)"
    if (( running < PROBE_PARALLEL )); then

    fi
```

### Topup 请求逻辑（第 1562-1699 行）

#### 请求体构建

```bash
# 使用 jq 构建请求体
jq -s --arg csv "$CLEAN_DELETE_STATUSES" '
  ($csv | split(",") | map(gsub("\\s"; "")) | map(select(test("^[0-9]{3}$")))) as $allow
  | map(select(
      (.replay_from_confidence == true)
      or (((.status_code // 0) | tostring) as $sc | ($allow | index($sc)) != null)
    ))
' "$REPORT_JSONL" > "$reports_tmp"

# 提取所有账号 ID
find "$ACCOUNTS_DIR" -name 'codex-*.json' \
  | sed -e 's#^.*/codex-##' -e 's#\.json$##' \
  | jq -R . | jq -s . > "$accids_tmp"

# 组装最终请求体
jq -n \
  --argjson target_pool_size "$request_target" \
  --slurpfile reports "$reports_tmp" \
  --slurpfile account_ids "$accids_tmp" \
  '{target_pool_size:$target_pool_size,reports:($reports[0] // []),account_ids:($account_ids[0] // [])}'
```

**关键点：**
1. **只上报失效账号**：仅包含状态码在 `CLEAN_DELETE_STATUSES` 中的账号（401, 429）
2. **包含所有账号 ID**：用于服务端去重和分配
3. **request_target 计算**：`TOTAL_HOLD_LIMIT - available_est`

#### 发送请求

```bash
curl -sS \
  --connect-timeout 10 \
  --max-time 180 \
  --retry 3 \
  --retry-all-errors \
  --retry-delay 3 \

  -H "X-User-Key: $USER_KEY" \
  -H "Content-Type: application/json" \

```

#### 响应处理（parse_topup_response 函数）

```bash
# 解析响应字段
ok=$(jq -r '.ok // false' "$resp_json")
auto_disabled=$(jq -r '.auto_disabled // false' "$resp_json")
abuse_auto_banned=$(jq -r '.abuse_auto_banned // false' "$resp_json")
accounts=$(jq -c '.accounts // []' "$resp_json")

# 下载账号
for acc in $(echo "$accounts" | jq -c '.[]'); do
  file_name=$(echo "$acc" | jq -r '.file_name')
  download_url=$(echo "$acc" | jq -r '.download_url // ""')
  auth_json=$(echo "$acc" | jq -r '.auth_json // ""')

  # 优先使用 auth_json
  if [[ -n "$auth_json" && "$auth_json" != "null" ]]; then
    echo "$auth_json" > "$ACCOUNTS_DIR/$file_name"
  elif [[ -n "$download_url" && "$download_url" != "null" ]]; then
    # 从 URL 下载
    curl -sS --connect-timeout 10 --max-time 60 \
      "$download_url" -o "$ACCOUNTS_DIR/$file_name"
  fi
done
```

## 与 Go 实现的对比

### 已实现的功能

✅ 增量闭环逻辑（第 1-6 轮）
✅ 并发探测（worker pool）
✅ 代理模式（auto/direct/proxy/mixed）
✅ Topup 请求（只上报失效账号）
✅ 下载账号（优先 auth_json）
✅ 删除失效账号
✅ 退出码处理

### 差异点

| 功能 | Bash 脚本 | Go 实现 | 说明 |
|------|----------|---------|------|
| 下载方式 | 串行 curl | 并发 HTTP | Go 实现性能更好 |
| 探测超时 | 单个请求超时 | 全局 + 单个超时 | Go 实现更精确 |
| 错误重试 | curl --retry | 手动重试 | 功能相同 |
| 日志输出 | echo | logger 包 | Go 实现更规范 |
| JSON 解析 | jq/osascript/python | encoding/json | Go 实现更可靠 |
| 并发控制 | jobs -pr + sleep | semaphore channel | Go 实现更高效 |

### 需要注意的细节

1. **request_target 计算**
   ```bash
   # Bash: 基于可用账号估算
   available_est=$((total - invalid))
   request_target=$((hold_limit - available_est))

   # Go: 应该保持一致
   availableEst := total - invalid
   requestTarget := holdLimit - availableEst
   ```

2. **增量范围文件**
   ```bash
   # Bash: 使用文件存储增量范围
   ITER_SCOPE_FILE="$ROOT_DIR/out/refill_iter_scope.txt"
   NEXT_SCOPE_FILE="$ROOT_DIR/out/refill_iter_scope_next.txt"

   # Go: 使用内存变量
   incrementalIDs []string
   ```

3. **探测报告格式**
